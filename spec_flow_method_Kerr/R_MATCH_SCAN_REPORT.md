# r_match 扫描测试报告

## 测试目的
针对高频区域 l=2, m=2 模式的较大偏差，测试改变匹配点 r_match 的影响。

## 测试参数
- 选取6个偏差最大的case (ω=0.4-0.5, a=0.0-0.9, l=2, m=2)
- r_match 范围：6.0, 8.0, 10.0, 12.0, 15.0, 20.0
- 固定：n_cheb=32, flow_eps=1e-6

## 主要发现

### 1. r_match 对结果无影响
所有测试case在 r_match = 6.0 到 20.0 范围内，计算结果完全一致（6位有效数字）。

示例 (a=0.7, ω=0.5, l=2, m=2):
- r_match = 6.0:  5.598843e-02
- r_match = 8.0:  5.598843e-02
- r_match = 20.0: 5.598843e-02
- 参考值: 2.120400e-02 (误差164%)

### 2. 数值收敛性测试
对最差case (a=0.7, ω=0.5) 测试：
- n_cheb: 32 → 48 → 64：结果不变
- flow_eps: 1e-6 → 1e-8 → 1e-10：结果不变

**结论：数值已完全收敛，偏差不是数值精度问题。**

### 3. 偏差模式总结
从 comparison_results.csv 分析：

**高精度区域 (相对误差 <1%):**
- 所有 l≥3 的模式
- 低频 (ω≤0.2) 的所有模式

**大偏差区域 (相对误差 >20%):**
- l=2, m=2 模式 + 高频 (ω≥0.3)
- 偏差随 ω 和 a 增大而增大
- 最大误差：a=0.7, ω=0.5, l=2, m=2 (164%)

## 可能原因分析

### 1. 近视界边界条件
当前使用：R ~ Δ² e^(-ik_H r*)

对于高频和高自旋，可能需要：
- 更高阶的 Δ 幂次修正
- 包含角动量耦合项
- 更精确的相位修正

### 2. λ 定义差异
参考文件的 spheroidal separation constant 可能：
- 使用不同的归一化
- 包含额外的修正项
- 与我们的定义有系统性差异

### 3. 参考方法的局限
MST方法在4个case失败（都是高频区），说明：
- 参考值在高频区也可能不够准确
- 我们的方法在这些区域反而更稳定
